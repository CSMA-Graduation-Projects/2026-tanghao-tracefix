# cfix_frontend

## 项目说明

这是这个项目当前代码对应的前端部分。它是一个用来承接“代码生成—执行—反馈—自动修复”流程的可视化界面，主要面向本地联调、实验查看和演示使用。

前端不是一个纯展示壳子。按照现在这版代码，它已经把登录、任务创建、代码生成、运行测试、自动修复、版本对比、历史任务、实验评估和系统本地设置放在同一套页面里，日常操作基本都围绕 `WorkBench` 展开。

## 技术栈

- Vue 3
- Vite
- Element Plus
- Vue Router
- Pinia
- Axios

## 当前页面

### 1. 登录页
路径：`/login`

作用比较直接：登录、拉取当前用户信息、保存 token。按照现在的后端实现，如果输入的用户名不存在，后端会自动创建一个新用户，再返回登录 token，所以第一次进入系统时不需要额外做注册页。

### 2. 工作台
路径：`/workbench`

这是当前前端的核心页面。按照现有代码，工作台主要负责这些事情：

- 管理会话和任务
- 填写题目描述、选择数据集、设置修复轮次
- 生成初始代码
- 运行当前版本
- 发起自动修复
- 查看结构化反馈、失败样例、修复计划、lesson、trace
- 查看版本时间线
- 查看版本代码和 diff
- 手动回滚到某个历史版本
- 导出任务 Markdown 报告

页面里已经做了不少状态联动，适合直接配合后端进行演示。

### 3. 历史任务页
路径：`/history`

这里主要用来查看已有任务、按标题搜索、删除任务、预览任务摘要，或者直接回到工作台继续打开某个任务。

### 4. 数据集页
路径：`/dataset`

这一页会调用后端的数据集接口，展示当前后端已经开放的数据集名称、显示名和基础说明。现在前后端对数据集的关系是：

- `custom`：函数级题库
- `class_bank`：类文件级题库
- `file_bank`：文件级题库

旧别名仍然兼容，但创建实验时主要使用上面这三个主数据集。

### 5. 实验评估页
路径：`/experiment`

实验页目前已经接了实验列表、创建实验、启动、停止、轮询进度、查看实验明细、查看报告和导出 Markdown 报告等流程。它更偏向批量实验和结果整理，不是单题工作台的替代页。

### 6. 系统设置页
路径：`/setting`

这页现在保存的是前端本地偏好，不会直接改后端配置。当前代码里已经接了这些默认项：

- 默认最大修复轮次
- 默认是否开启 Trace
- 默认是否开启 Lesson
- 默认是否通过即停

这些值会存到浏览器本地存储里，后面进入工作台时会自动读取。

## 环境变量

前端当前只用到一个核心环境变量：

```env
VITE_API_BASE_URL=http://127.0.0.1:8000/api/v1
```

如果你的后端端口或地址不是这个值，改掉 `.env` 即可。

## 安装与启动

### 1. 安装依赖

```bash
npm install
```

### 2. 准备环境变量

把 `.env.example` 复制成 `.env`，再确认接口地址：

```bash
cp .env.example .env
```

Windows PowerShell 可以用：

```powershell
Copy-Item .env.example .env
```

### 3. 启动开发环境

```bash
npm run dev
```

默认会启动 Vite 开发服务器。实际地址以终端输出为准，常见是：

```text
http://127.0.0.1:5173
```

### 4. 生产构建

```bash
npm run build
```

### 5. 本地预览构建结果

```bash
npm run preview
```

## 登录与鉴权说明

前端请求统一走 `src/utils/req.js`。当前这层封装已经做了这些事情：

- 自动拼接 `VITE_API_BASE_URL`
- 自动带上 `Authorization` 请求头
- 统一处理后端 `{ code, msg, data }` 返回结构
- 统一处理 401 / 403 / 404 / 500
- 登录失效时清空本地 token 并跳转到登录页

登录信息目前保存在浏览器本地存储中，至少会用到：

- `token`
- `token_type`
- `username`

## 常用联调流程

第一次把前后端一起跑起来时，按下面这个顺序会比较顺手：

1. 启动后端服务
2. 启动前端服务
3. 打开 `/login`
4. 输入用户名和密码登录
5. 进入工作台后先新建会话
6. 创建任务并生成初始代码
7. 手动运行一次测试
8. 再尝试自动修复、查看版本和反馈

如果你是想验证实验页，可以在前面的流程走通之后再去 `/experiment` 创建批量实验。

## 当前前端和后端的配合方式

这套前端默认配合同项目下的 FastAPI 后端使用。当前已经接入的主要接口分组有：

- `auth`：登录、获取当前用户、退出登录
- `chat`：会话和消息
- `task`：任务创建、更新、生成、执行、自动修复、测试用例、plan、lesson
- `run`：运行结果、结构化反馈、trace、单用例详情
- `ver`：版本详情、版本代码、diff、回滚
- `exp`：实验列表、创建、启动、停止、明细、报告、图表
- `data`：数据集列表和详情
