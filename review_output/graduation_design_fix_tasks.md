# 毕业设计整改任务清单

## 必须立即修改

### 1. 修复一键部署文件名错误

- 问题：`docker-compose.yml` 和根 README 引用 `cfix/cfix_schema.sql`，但本地实际只有 `cfix/cfix_db.sql`。
- 修改目标：保证 `docker compose up --build` 能正确初始化 MySQL。
- 建议做法：统一文件名引用；验证数据库容器启动后能导入 `cf_user`、`cf_task`、`cf_run` 等表。
- 影响位置：代码 / 部署 / README

### 2. 补充真实测试，替换占位测试

- 问题：`tests/test_placeholder.py` 只有 `assert True`，不能证明系统功能可用；`req.txt` 也没有 pytest。
- 修改目标：让测试章节和代码仓库中的测试相互对应。
- 建议做法：增加 pytest 依赖；至少补登录、任务创建、测试用例规范化、代码执行反馈、自动修复主流程、实验统计这 6 类测试。
- 影响位置：代码 / 论文

### 3. 修正实验统计异常

- 问题：`reg_qwen.md` 出现“最终通过率 200.00%”，实验数据口径明显错误。
- 修改目标：所有通过率、提升率、修复成功率都必须在合理范围内，并能追溯到原始结果。
- 建议做法：检查 `scripts/reg_qwen.py`、`scripts/exp_cmp.py` 的统计公式；重新生成 Markdown、CSV、JSON；同步更新论文和 PPT。
- 影响位置：代码 / 论文 / PPT

### 4. 核实 Trace、Lesson、Rollback 的实验支撑

- 问题：论文和 PPT 强调完整链路优势，但部分结果文件显示 `trace 非空题目数：0`。
- 修改目标：明确 Trace 是否真实参与了实验，以及参与后带来的效果。
- 建议做法：补充 trace 有效样例；如果 trace 没有触发，要在论文中解释原因，并弱化“Trace 带来明显提升”的绝对表述。
- 影响位置：代码 / 论文 / PPT

### 5. 确认部署平台能完整演示

- 问题：当前只能确认 EdgeOne 站点头信息可达，未能实际验证页面、登录和核心流程。
- 修改目标：答辩时打开链接后能完成至少一条主流程展示。
- 建议做法：检查前端静态资源、后端 API 地址、CORS、演示账号、token 有效期；准备本地备用演示命令。
- 影响位置：部署 / README / PPT

### 6. 统一论文、PPT 与代码中的函数/模块名称

- 问题：PPT 中出现 `task_svc.auto_repair()`、`run_svc.execute()` 等名称，和代码实际的 `auto_fix()`、`run_version()` 不一致。
- 修改目标：答辩材料中的关键函数名能在代码中直接找到。
- 建议做法：把 PPT 和论文中的函数名改为代码实际名称，或统一改成中文概念描述。
- 影响位置：论文 / PPT

## 建议尽快修改

### 7. 补充答辩演示脚本和用户手册

- 问题：README 有运行说明，但缺少“答辩时怎么演示”的可执行流程。
- 修改目标：学生可以按固定步骤稳定展示系统特色。
- 建议做法：新增一份演示说明，写清登录账号、示例题目、测试用例、生成代码、运行失败、自动修复、查看 diff、查看实验报告的步骤。
- 影响位置：README / docs / PPT

### 8. 强化论文测试章节证据

- 问题：论文写了单元测试、集成测试、系统测试、性能测试，但本地测试文件无法支撑这些结论。
- 修改目标：测试章节从“展示性描述”变成“可复查证据”。
- 建议做法：补测试命令、测试文件路径、测试结果截图、失败用例与修复前后对比表。
- 影响位置：论文 / 代码

### 9. 处理 `stream.py` placeholder

- 问题：后端 `stream.py` 仍返回固定 placeholder 数据。
- 修改目标：避免材料中出现未实现功能。
- 建议做法：若答辩不展示实时进度，就不要在论文/PPT强调流式进度；若要展示，改成真实任务状态事件流。
- 影响位置：代码 / 论文 / PPT

### 10. 补充部署和模型配置排错说明

- 问题：真实模型调用依赖 `.env` 配置，当前说明不足以保证其他人复现。
- 修改目标：检查老师或答辩现场能判断系统如何接入真实模型。
- 建议做法：写清 `LLM_ENABLE`、provider、model、base_url、api_key、strict 模式、失败降级逻辑和无 key 时的演示方案。
- 影响位置：README / 部署 / 论文

### 11. 完善实验原始材料归档

- 问题：本地可见的原始实验输出不足以完整支撑论文/PPT中三类实验结论。
- 修改目标：每个实验结论都有对应文件路径和原始结果。
- 建议做法：按函数级、类文件级、文件级分别归档 Markdown、CSV、JSON；在论文中标注样本数、模型、轮次、配置。
- 影响位置：论文 / PPT / 代码

## 可选优化

### 12. 增加 CI 或本地 smoke 脚本

- 问题：当前缺少一条命令验证后端、前端和数据库基础可用。
- 修改目标：减少答辩前临时部署失败风险。
- 建议做法：增加 `scripts/smoke_backend.sh` 或 README 中的 smoke 步骤，检查 `/docs`、登录、数据集接口、任务创建接口。
- 影响位置：代码 / README

### 13. 优化 PPT 的实验页表达

- 问题：实验页偏结果展示，缺少指标口径和配置说明。
- 修改目标：让评委快速看懂实验怎么跑、结论从哪里来。
- 建议做法：每类实验页加上模型、样本数、最大轮次、三种配置说明和关键指标定义。
- 影响位置：PPT

### 14. 扩大实验规模

- 问题：当前论文也承认每类题库规模较小，结论说服力有限。
- 修改目标：增强实验可信度。
- 建议做法：在时间允许时增加更多题目或引入公开数据集；至少补充多次运行均值，降低单次模型输出波动影响。
- 影响位置：代码 / 论文 / PPT

### 15. 明确 safe_exec 与 Docker 的边界

- 问题：系统同时支持 safe_exec 和 Docker，但 README、论文和 PPT 对演示环境差异说明还不够清楚。
- 修改目标：避免评委误以为线上一定是强隔离 Docker 沙箱。
- 建议做法：说明本地联调默认用什么，安全演示推荐用什么，Docker 模式需要哪些宿主机条件。
- 影响位置：README / 论文 / PPT
